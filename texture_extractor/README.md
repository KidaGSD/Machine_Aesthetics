# Emotion-Driven Parametric Lamp Cover System - Texture Extractor

This component of the project handles texture processing and parametric design generation based on pre-computed audio emotion data. It takes an audio emotion timeline (Valence, Arousal, Dominance over time) and a collection of textures, classifies the textures by emotion, generates depth maps, maps audio emotions to relevant textures, blends them over time, and finally generates parameters suitable for driving a parametric lamp design in Rhino/Grasshopper.

## System Overview

The pipeline consists of several stages, orchestrated by `main.py` and configured via `config/pipeline_config.yaml`:

1.  **(Preprocessing - Optional, run separately)**
    *   **Texture Classification**: Classifies input textures (e.g., from DTD dataset) by emotion using CLIP (`src/texture_classifier.py`). Outputs a list of high-confidence images and detailed results.
    *   **Depth Map Generation**: Generates depth maps for the high-confidence textures using MiDaS (`src/depth_generator.py`).
2.  **Texture Mapping**: Maps emotions from the input audio VAD timeline to the high-confidence textures based on VAD ranges defined in `config/emotion_mapping_vad.json` (`src/texture_mapper.py`). Outputs a `texture_timeline.json`.
3.  **Texture Blending**: Blends textures according to the timeline, generating sequences of blended textures and corresponding displacement maps (`src/texture_blender.py`). Outputs blended images, displacement maps, and `animation_data.json` (downsampled displacement grids).
4.  **Parametric Design**: Generates parametric design parameters (JSON files) based on the `animation_data.json`, suitable for use in Rhino/Grasshopper (`src/parametric_generator.py`). Also generates preview images and example Rhino scripts.
5.  **(Evaluation - Optional)**: Analyzes and visualizes results from various stages (`src/evaluation_module.py`).

## Installation

### Prerequisites

- Python 3.8+
- CUDA-compatible GPU (recommended for CLIP and MiDaS)
- [Rhino 7](https://www.rhino3d.com/) with [Grasshopper](https://www.grasshopper3d.com/) (for using the generated parametric designs)
- Kaggle API configured (for downloading DTD dataset via `setup_dtd.py`)

### Setup

1.  **Clone Repository**: If you haven't already, clone the main project repository.
2.  **Navigate**: Change directory into `texture_extractor/`.
    ```bash
    cd path/to/Machine_Aesthetics/texture_extractor
    ```
3.  **Install Dependencies**: Create a virtual environment (recommended) and install required packages.
    ```bash
    python -m venv venv
    # Activate environment (Windows)
    venv\Scripts\activate 
    # Activate environment (macOS/Linux)
    # source venv/bin/activate 
    pip install -r requirements.txt
    # Special CLIP install (if needed, might be handled by requirements)
    # pip install git+https://github.com/openai/CLIP.git 
    ```
4.  **Download DTD Dataset**: Run the setup script to download the Describable Textures Dataset.
    ```bash
    python setup_dtd.py 
    ```
    This will download and extract the dataset into `texture_extractor/data/dtd/images/`.

## Configuration

The pipeline is configured using `config/pipeline_config.yaml`. Key settings include:

-   `original_textures_dir`: Path to the input texture images (e.g., `data/dtd/images`).
-   `audio_timeline_path`: **Crucially**, provide the path to the CSV file containing the pre-computed Valence-Arousal timeline generated by the `audio_analysis` component.
-   `emotion_mapping_path`: Path to the JSON defining VAD ranges for emotion labels.
-   `results_base_dir`: Base directory for all outputs.
-   Settings for classification, depth generation, blending, and parametric design (thresholds, model types, sizes, scales, etc.).
-   `device`: Set to "cuda" or "cpu", or leave as `null` for auto-detection.

**Before running the main pipeline, ensure `audio_timeline_path` in the config points to a valid CSV file.**

## Usage

### 1. Preprocessing Textures (Optional First Step)

If you need to classify textures and generate depth maps:

```bash
# Run both classification and depth generation
python preprocess_textures.py --config config/pipeline_config.yaml 

# Skip classification if already done
# python preprocess_textures.py --config config/pipeline_config.yaml --skip_classification

# Skip depth generation if already done
# python preprocess_textures.py --config config/pipeline_config.yaml --skip_depth 
```
This script uses settings from the `classification` and `depth_generation` sections of the config file and saves results (high-confidence list, depth maps) to the configured output directories within `results/`.

### 2. Running the Main Pipeline

Once textures are preprocessed and the `audio_timeline_path` in the config is set correctly:

```bash
# Run the full pipeline (mapping, blending, design)
python main.py --config config/pipeline_config.yaml

# --- Or run specific stages ---

# Run only texture mapping
# python main.py --config config/pipeline_config.yaml --skip_blending --skip_design

# Run mapping and blending
# python main.py --config config/pipeline_config.yaml --skip_design 

# Run blending and design (requires existing texture_timeline.json)
# python main.py --config config/pipeline_config.yaml --skip_mapping 
```
The `run_pipeline.bat` (Windows) and `run_pipeline.sh` (macOS/Linux) scripts are simple wrappers to execute `main.py`.

### 3. Using Parametric Designs

-   The pipeline generates JSON parameter files in `results/parametric/`.
-   It also generates example Rhino Python scripts (`*_rhino.py`) alongside the JSON files.
-   You can:
    -   Load the JSON parameters into a compatible Grasshopper definition (like the one potentially in `grashopper/ghtest02.gh`).
    -   Run the generated `*_rhino.py` script directly within Rhino's Python script editor (`_EditPythonScript`).
